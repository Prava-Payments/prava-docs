## Endpoint

```
GET /v1/sessions/:sessionId/payment-result
```

| Environment | Full URL |
|-------------|----------|
| **Sandbox** | `https://sandbox.api.prava.space/v1/sessions/:sessionId/payment-result` |
| **Production** | `https://api.prava.space/v1/sessions/:sessionId/payment-result` |

## Authentication

Bearer token using your **secret key**:

```
Authorization: Bearer sk_test_xxx
```

## Path Parameters

<ParamField path="sessionId" type="string" required>
  The session ID returned from [Create Session](/api-reference/create-session)
</ParamField>

## Response

<ResponseField name="session_id" type="string">
  The session identifier
</ResponseField>

<ResponseField name="order_id" type="string | null">
  The internal order identifier
</ResponseField>

<ResponseField name="status" type="string">
  Overall session status: `pending`, `awaiting_result`, `completed`, or `failed`
</ResponseField>

<ResponseField name="transactions" type="array">
  Array of transactions within this session

  <Expandable title="Transaction object">
    <ResponseField name="txn_id" type="string">
      Unique transaction identifier
    </ResponseField>

    <ResponseField name="status" type="string">
      Transaction status: `pending`, `awaiting_result`, `completed`, or `failed`
    </ResponseField>

    <ResponseField name="line_items" type="array">
      Transaction line items with payment credentials

      <Expandable title="Line item object">
        <ResponseField name="txn_ref_id" type="string">
          Transaction line item reference ID (used in [Report Status](/api-reference/report-status))
        </ResponseField>

        <ResponseField name="merchant_name" type="string | null">
          Merchant name
        </ResponseField>

        <ResponseField name="merchant_url" type="string | null">
          Merchant URL
        </ResponseField>

        <ResponseField name="total_amount" type="string">
          Total amount for this line item
        </ResponseField>

        <ResponseField name="status" type="string">
          Line item status
        </ResponseField>

        <ResponseField name="token" type="string | null">
          Virtual card number (network token) for checkout. Only present when status is `awaiting_result`.
        </ResponseField>

        <ResponseField name="dynamic_cvv" type="string | null">
          Single-use CVV for checkout. Only present when status is `awaiting_result`.
        </ResponseField>

        <ResponseField name="expiry_month" type="string | null">
          Token expiration month
        </ResponseField>

        <ResponseField name="expiry_year" type="string | null">
          Token expiration year
        </ResponseField>

        <ResponseField name="products" type="array">
          Products in this line item

          <Expandable title="Product object">
            <ResponseField name="product_ref_id" type="string">
              Internal product reference ID
            </ResponseField>

            <ResponseField name="external_product_id" type="string | null">
              Your external product identifier
            </ResponseField>

            <ResponseField name="name" type="string">
              Product name/description
            </ResponseField>

            <ResponseField name="unit_price" type="string">
              Unit price
            </ResponseField>

            <ResponseField name="quantity" type="number">
              Quantity
            </ResponseField>
          </Expandable>
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="error" type="object">
      Present only when the transaction has failed

      <Expandable title="Error object">
        <ResponseField name="code" type="string">
          Error code
        </ResponseField>

        <ResponseField name="message" type="string">
          Human-readable error message
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Example

<Tip>Swap the host for production: use `https://api.prava.space` instead of `https://sandbox.api.prava.space`.</Tip>

<CodeGroup>
```bash cURL — Sandbox
curl https://sandbox.api.prava.space/v1/sessions/ses_a1b2c3d4/payment-result \
  -H "Authorization: Bearer sk_test_xxx"
```

```bash cURL — Production
curl https://api.prava.space/v1/sessions/ses_a1b2c3d4/payment-result \
  -H "Authorization: Bearer sk_live_xxx"
```

```typescript Node.js
const BASE_URL = 'https://sandbox.api.prava.space'; // or https://api.prava.space

const result = await fetch(
  `${BASE_URL}/v1/sessions/ses_a1b2c3d4/payment-result`,
  {
    headers: {
      'Authorization': `Bearer ${SECRET_KEY}`,
    },
  }
).then(res => res.json());
```
</CodeGroup>

## Error Responses

| Status | Code | Cause |
|--------|------|-------|
| 401 | `AUTH_1001` | Invalid API key |
| 401 | `AUTH_1002` | Missing or invalid Authorization header |
| 404 | `NOT_FOUND` | Session not found or doesn't belong to your merchant account |

### Error Response Format

```json
{
  "error": {
    "code": "NOT_FOUND",
    "message": "Session not found"
  }
}
```

## Notes

- This is a **polling endpoint** — call it after the user completes authentication and intent registration to retrieve payment credentials.
- The `token` and `dynamic_cvv` fields contain the virtual card credentials your agent uses at checkout.
- Prava performs a lazy mandate expiry check on every request — expired mandates will be reflected in the status.
- After using the credentials at checkout, report the outcome via [Report Status](/api-reference/report-status).
